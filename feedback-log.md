# Feedback Log

## Project Direction

The project idea was selected after considering multiple Project 2 options. DeadlinePilot was chosen because it clearly demonstrates agentic behavior through task parsing, feasibility checking, triage, risk detection, and reviewer approval.

## Project 1 Carryover Feedback

Project 1 feedback emphasized the need for stronger evaluation, especially cases that reveal failure instead of only showing successful outputs.

For Project 2, DeadlinePilot includes an overload failure case and a messy input case. These are designed to test whether the system can identify unrealistic workloads and incomplete information.

## Current Feedback

The current draft is working but incomplete. The app demonstrates the main workflow, but messy-input handling needs improvement in the next version.
